    🤖 Pioneer P3DX
    Robot móvil diferencial

    Robot móvil diferencial muy utilizado en investigación y docencia. En CoppeliaSim resulta ideal para aprender movimiento, sensores y control básico con Python.

    FabricanteAdept MobileRobots
    Usado enpractica01, practica02, practica03

    Características principales:

    • Dos ruedas motrices
    • Rueda loca trasera
    • 16 sensores ultrasónicos
    • Modelo sencillo para primeras prácticas
    • Compatible con control mediante Python
    Consejo para el profesor: Es un robot ideal para empezar porque permite explicar avance, retroceso y giro usando únicamente dos motores.
    Python: sim.getObject()

    Permite obtener la referencia interna de un objeto de la escena a partir de su ruta.

    Sintaxis:

    sim.getObject('/ruta/del/objeto')

    Devuelve: Identificador interno del objeto dentro de CoppeliaSim.

    Errores frecuentes:

    • Ruta incorrecta del objeto.
    • El objeto no existe en la escena.
    • Usar el nombre visible pero no la ruta completa.

    Ejemplo:

    motor_izquierdo = sim.getObject('/PioneerP3DX/leftMotor')
    📡 Sensor ultrasónico
    Sensor de proximidad

    Sensor que mide la distancia aproximada a un obstáculo utilizando una señal simulada. En el Pioneer P3DX se utiliza para detectar paredes, objetos y obstáculos cercanos.

    Características principales:

    • Permite detectar obstáculos
    • No proporciona imagen, solo distancia o detección
    • Es útil para navegación básica
    • Se puede usar para evitar colisiones

    Usado en: practica07, practica08

    Consejo para el profesor: Es ideal para explicar que un robot no ve el mundo como una persona: solo recibe datos numéricos de sus sensores.
    ⚙️ sim.startSimulation()
    Comando de simulación

    Inicia la simulación activa en CoppeliaSim. A partir de este momento comienza a avanzar el tiempo simulado y se activan scripts, sensores, motores y el motor físico.

    Sintaxis:

    ```python sim.startSimulation()

    Errores frecuentes:

    • Intentar controlar motores antes de iniciar la simulación.
    • Ejecutar varias veces la simulación sin detenerla correctamente.
    • No esperar unos instantes a que la simulación arranque.

    Usado en: practica01, practica02

    Consejo para el profesor: Conviene explicar que iniciar la simulación no es lo mismo que conectar Python: primero Python se conecta al simulador y después se ordena iniciar la escena.
    📚 Robot diferencial
    Robótica móvil

    Robot móvil que se desplaza mediante dos ruedas motrices independientes. Para avanzar, ambas ruedas giran en el mismo sentido. Para girar, las ruedas giran a velocidades distintas o incluso en sentidos opuestos.

    Relacionado con: PioneerP3DX, setJointTargetVelocity, motor

    Consejo para el profesor: Es uno de los conceptos más importantes de la primera parte del curso. Conviene explicarlo con un dibujo en la pizarra antes de programarlo.
